# style-creator — 나만의 문체 스킬 생성기

이 스킬은 사용자와의 인터뷰를 통해 **그 사람만의 문체 스킬**을 만들어준다.
결과물은 `SKILL.md` + 플랫폼별 `references/*.md` 파일 세트로, 생성 즉시 Claude에서 사용 가능하다.

---

## 작동 흐름

```
0단계  스킬 이름 확인
1단계  플랫폼 선택  ← 가장 먼저
2단계  인터뷰 (레이어 1~3)
3단계  각 레이어 초안 제시 → 사용자 확인·수정
4단계  파일 생성 및 전달
```

---

## 실행 지침

### 시작 멘트

> "지금부터 나만의 문체 스킬을 만들어드릴게요.
> 먼저 두 가지만 확인할게요."

### 0단계 — 스킬 이름 확인

> "만들 스킬 이름을 정해주세요. 기본값은 `mystyle`이에요.
> 이미 `mystyle`을 사용 중이라면 다른 이름(예: `writingstyle`, `my-voice`)으로 하시면 기존 스킬이 유지돼요."

사용자가 이름을 정하면 이후 모든 파일 생성에 해당 이름을 사용한다.

### 1단계 — 플랫폼 선택

`references/interview.md`의 **플랫폼 선택 섹션**을 읽고 진행한다.

- 지원 목록에 없는 플랫폼이 언급되면 → `references/platform-mapping.md`를 읽고 처리한다.
- 플랫폼이 3개 이상이면 → "가장 자주 쓰시는 것 2~3개부터 먼저 만들고 나머지는 나중에 추가하는 방식도 괜찮아요"라고 안내한다.

### 2~3단계 — 인터뷰 진행

`references/interview.md`의 **레이어 1~3 섹션**을 순서대로 읽고 진행한다.

**인터뷰 원칙**
- 한 번에 질문 2개 이하
- 답변이 짧거나 모호하면 한 번만 구체화 질문
- 각 레이어가 끝나면 Claude가 초안을 먼저 작성해서 보여줌. 사용자가 직접 채우게 하지 않음
- 사용자가 "됐어요" / "그냥 만들어줘"라고 하면 → 지금까지 답변으로 즉시 생성

### 4단계 — 파일 생성

`references/template.md`를 읽고 아래 파일을 생성한다.

1. `{스킬명}/SKILL.md` — 공통 원칙 + 매체 라우팅
2. `{스킬명}/references/{플랫폼}.md` — 선택한 플랫폼 수만큼

생성 후 파일 구조를 요약해서 보여주고, 설치 방법을 안내한다.

## 레퍼런스 파일

| 파일 | 용도 |
|---|---|
| `references/interview.md` | 플랫폼 선택 질문 + 레이어별 인터뷰 질문 |
| `references/template.md` | SKILL.md + 플랫폼 파일 템플릿 |
| `references/platform-mapping.md` | 미지원 플랫폼 처리 규칙 |
