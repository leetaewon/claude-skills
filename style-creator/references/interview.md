# 인터뷰 질문 목록

style-creator가 순서대로 읽고 진행하는 질문 목록이다.
질문을 한꺼번에 던지지 않는다. 단계별로 진행하고, 각 레이어가 끝나면 Claude가 초안을 제시한다.

## 레이어 1 — 나는 누구인가 (XYZ)

목적: BI 필터의 재료를 뽑는다. "이 사람이 어떤 사람인지"가 글의 기저에 흐르도록 한다.

**Q1-1.** (What + Who)
> "어떤 일을 하시나요? 주로 어떤 분들에게 도움을 드리고 있나요?"

**Q1-2.** (Why)
> "이 일을 하는 이유나, 글을 쓸 때 가장 중요하게 생각하는 가치가 있다면요?"

### 레이어 1 초안 제시 형식

답변을 받으면 Claude가 아래 형식으로 초안을 만들어 보여준다.

```
[BI 필터 초안]
- 이 글은 {독자}를 위한 글입니다.
- 핵심 가치: {Why에서 추출한 1~2가지}
- 이 글에서 절대 하지 않는 것: {가치와 반대되는 표현 패턴 1~2가지 제안}
```

확인 후 수정하면 레이어 2로 넘어간다.

## 레이어 2 — 어떻게 풀어가는가 (서사 방식)

목적: 글의 기본 전개 방식을 결정한다. 어휘나 어미보다 더 강한 개성을 만드는 레이어다.

**Q2-1.**
> "글을 쓸 때 결론을 먼저 말하는 편인가요, 아니면 과정을 따라가면서 보여주는 편인가요?"

**Q2-2.**
> "독자에게 어떤 인상을 남기고 싶으세요? 가장 가까운 것을 골라주세요.
> ① 친근함 — 옆에 있는 선배 같은 느낌
> ② 전문성 — 신뢰할 수 있는 전문가 느낌
> ③ 동료감 — 나도 같은 고민을 해봤어요
> ④ 영감 — 새로운 관점을 열어주는 느낌"

### 서사 유형 매핑

| 조합 | 기본 서사 패턴 |
|---|---|
| 결론 먼저 + 전문성 | 두괄식: 핵심 → 근거 → 적용 |
| 결론 먼저 + 영감 | 반전형: 질문 제기 → 뒤집기 → 새 관점 |
| 과정 중심 + 친근함 | 과정형: 상황 → 해봤더니 → 이렇게 됐어요 |
| 과정 중심 + 동료감 | 공감형: 독자 상황 짚기 → 나도 그랬어요 → 같이 해봐요 |

### 레이어 2 초안 제시 형식

```
[서사 패턴 초안]
기본 흐름: {선택된 서사 유형}
글 구조 예시:
  첫 문단 — {시작 방식}
  → 본론 — {전개 방식}
  → 마무리 — {마무리 방식}
```

## 레이어 3 — 어떻게 표현하는가 (표현 선호도)

목적: 공통 원칙의 "유지할 것 / 절대 쓰지 말 것" 섹션을 채운다.

**Q3-1.** (어조)
> "존댓말로 쓰시나요, 반말로 쓰시나요? 플랫폼마다 다르게 쓰신다면 그것도 말씀해주세요."

**Q3-2.** (금지 표현)
> "글에서 절대 쓰고 싶지 않은 표현이나 말투가 있나요?
> 예: '지금 안 하면 늦어요', '완벽 가이드', 과도한 이모지, 너무 딱딱한 문어체 등"

**Q3-3.** (부가 선호도)
> "아래 항목도 간단히 알려주시면 반영할게요.
> - 이모지: 자주 쓰는 편 / 가끔 / 안 씀
> - 문장 길이: 짧고 간결하게 / 길고 설명적으로
> - 개인 경험 삽입: 자주 / 가끔 / 거의 안 함"

### 레이어 3 초안 제시 형식

```
[표현 원칙 초안]

유지할 것:
- {존댓말/반말}
- {서사 방식에서 파생된 표현 습관}
- {사용자가 선호한다고 밝힌 것}

절대 쓰지 말 것:
- {사용자가 명시한 금지 표현}
- {BI 필터에서 파생된 금지 패턴}
```
