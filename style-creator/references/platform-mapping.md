# 플랫폼 매핑 규칙

지원 목록에 없는 플랫폼이 언급되면 이 파일을 읽고 처리한다.
처리 방식은 두 레벨로 나뉜다.

---

## 레벨 1 — 자동 매핑 (사용자 확인 없이 처리)

유사성이 명확한 경우 안내만 하고 바로 적용한다.

> "○○는 □□와 비슷한 형식이라 `{파일명}`을 기반으로 만들게요."

| 언급된 플랫폼 | 매핑 대상 | 이유 |
|---|---|---|
| 스레드(Threads) | instagram.md | 단문·세로 피드 형식 동일 |
| X (트위터) | instagram.md | 단문 중심, 해시태그 유사 |
| 카카오스토리 | facebook.md | 국내 지인 네트워크 SNS |
| 네이버 블로그 | blog.md | 장문 블로그 형식 동일 |
| 미디엄(Medium) | blog.md | 장문 아티클 형식 동일 |
| 노션 공개 페이지 | blog.md | 문서형 공개 글 |
| 카카오 채널 포스트 | newsletter.md | 구독자 대상 발행 형식 유사 |
| 네이버 포스트 | blog.md | 아티클 형식 유사 |
| 핀터레스트 설명글 | instagram.md | 이미지 중심 짧은 설명 |

---

## 레벨 2 — 사용자 확인 후 처리

자동 판단이 애매한 경우 아래 안내를 보여주고 선택하게 한다.

> "'{플랫폼명}'은 제가 준비한 목록에 없는 채널이에요.
> 두 가지 방법 중 선택해주세요.
> ① 가장 비슷한 '{유사 플랫폼}'을 기반으로 만들기
> ② 이 채널 전용으로 새로 만들기 (특성을 조금 더 알려주시면 돼요)"

| 상황 | 유사 플랫폼 제안 | 비고 |
|---|---|---|
| 사내 위키·사내 블로그 | blog.md | 단, 독자가 외부인이 아님을 명시 |
| 포럼·커뮤니티 게시판 | facebook.md | 대화체·짧은 호흡 |
| 팟캐스트 스크립트 | youtube.md | 음성 중심 구어체 동일 |
| 강의 교안·PPT 노트 | book.md | 구조화된 장문 형식 유사 |
| 카드뉴스 본문 | instagram.md | 짧은 문장, 슬라이드 단위 |
| 이름을 모르는 플랫폼 | — | 형식을 묻고 판단 |

---

## 형식을 모를 때 물어볼 질문

사용자가 플랫폼 이름만 말하고 형식을 알 수 없을 때:

> "'{플랫폼명}'에서 주로 어떤 형식으로 글을 쓰시나요?
> 예: 짧은 글 / 긴 글, 이미지 중심 / 텍스트 중심, 독자가 구독자 / 불특정 다수"

답변을 바탕으로 레벨 1 매핑표에서 가장 유사한 파일을 선택한다.

---

## 처리 후 파일명 규칙

- 자동 매핑: 매핑된 원본 파일명 사용 (예: instagram.md)
- 새로 만드는 경우: 사용자가 부르는 이름을 소문자·하이픈으로 변환
  - 예: "카카오채널" → `kakao-channel.md`
  - 예: "사내 위키" → `internal-wiki.md`
