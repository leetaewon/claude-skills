# 생성 템플릿

style-creator가 인터뷰 완료 후 이 템플릿을 기반으로 파일을 생성한다.
`{중괄호}` 항목을 인터뷰 결과로 채워서 출력한다.

---

## SKILL.md 템플릿

```markdown
---
name: {스킬명}
description: {작성자 이름 또는 닉네임}의 문체로 글을 작성하는 Skill. 사용자가 특정 매체({지원 플랫폼 나열})를 명시하거나, "내 스타일로", "{닉네임} 스타일로", "{스킬명}로" 글을 써달라고 요청할 때 반드시 이 Skill을 사용한다. 매체별 문체 가이드를 자동으로 로드하여 적용한다.
---

# {스킬명} — {작성자 이름} 문체 Skill

## 작동 방식

1. 사용자가 **매체를 명시**하면 → 해당 references 파일을 읽고 적용
2. 매체가 **불분명**하면 → 아래 매체 목록을 안내하고 선택을 요청
3. 글감(주제/소재)이 없으면 → 요청 후 작성

## 지원 매체 목록

| 사용자 표현 | 로드할 파일 |
|---|---|
{플랫폼 행 목록}

## 매체별 파일 로드 규칙

매체가 결정되면 **해당 파일 하나만** 읽는다. 여러 파일을 동시에 읽지 않는다.

## 공통 기본 원칙 (모든 매체 공통)

**BI 필터**
{BI 필터 내용 — 레이어 1 결과}

**서사 패턴**
{서사 패턴 내용 — 레이어 2 결과}

**유지할 것**
{유지할 것 목록 — 레이어 3 결과}

**절대 쓰지 말 것**
{절대 쓰지 말 것 목록 — 레이어 3 결과}

---

*{스킬명} | {작성자 이름} 문체 Skill | v1.0 | {생성일}*
```

## 플랫폼별 references 파일 템플릿

### 공통 구조

```markdown
# {플랫폼명} 문체 가이드

## Claude 지시 프롬프트

{작성자 이름}의 {플랫폼명} 문체로 써줘.

기본 스타일:
{레이어 2 서사 패턴을 이 플랫폼에 맞게 적용}

BI 필터:
{레이어 1 핵심 가치와 금지 패턴}

{플랫폼명} 규칙:
{아래 플랫폼별 기본값 참고하여 채움}

## 글 구조

{서사 패턴 + 플랫폼 특성을 결합한 흐름}

## 주의사항
{플랫폼별 주의사항}
```

## 플랫폼별 기본값

인터뷰에서 언급되지 않은 항목은 아래 기본값을 적용한다.

### 블로그 (blog.md)
- 소제목 2~4개로 구조 잡기
- 첫 단락: 독자 상황 공감 or 이 글을 쓰게 된 계기
- 마무리: 정리보다 "앞으로 이렇게 해볼 생각" or 독자 질문
- 번호 나열형 구조 지양
- 분량: 1,500~3,000자

### 브런치 (brunch.md)
- 소제목 없이 흐르는 산문 가능
- 에세이 감성 허용: 개인 경험 중심
- 첫 문장이 글의 분위기를 결정
- 분량: 1,000~2,500자

### 뉴스레터 (newsletter.md)
- 한 메일에 하나의 발견만
- 인사 없이 바로 상황으로 시작
- 마지막은 독자에게 던지는 질문 하나
- 소제목·번호 매기기 없이 산문체
- 모바일 가독성: 단락 짧게
- 링크·CTA 하나만

### 인스타그램 (instagram.md)
- 첫 줄이 "더 보기" 전 전부 — 공감 또는 질문으로 시작
- 줄바꿈 적극 활용
- 이모지: 사용자 선호에 따라 (기본값: 1~3개)
- 해시태그: 5~10개
- 분량: 300~600자

### 페이스북 (facebook.md)
- 총 10~15줄 이내
- 번호·소제목 없이 흐르는 산문
- 마지막은 댓글 유도하는 열린 질문
- "더 보기" 접힘 전 3~4줄이 핵심
- 해시태그 1~2개

### 링크드인 (linkedin.md)
- 첫 줄: 인사이트 한 줄 or 반전 질문
- 전문성과 경험을 결합한 서술
- 줄바꿈으로 가독성 확보
- 해시태그 3~5개
- 분량: 150~400자 또는 500자 이상 롱폼

### 유튜브 스크립트 (youtube.md)
- 첫 15초: 시청 이유 제시 (후킹)
- 구어체 강화: 문어체 표현 금지
- 시각 자료 표시: [자막], [화면 전환] 명시
- 인트로 → 본론 3개 내외 → 아웃트로 구조
- 아웃트로: 구독·좋아요 자연스럽게 연결

### 책·원고 (book.md)
- 챕터 오프닝: 상황·질문 제시로 시작
- 실제 경험과 가상 예시 구분 ("예를 들면~"으로 명시)
- 소제목 챕터당 3~5개
- 단락 4~6줄
- 교재형 선언·핵심 정리 박스 과용 금지

