# 리드마그넷 제작 가이드

## 포지셔닝

- 웨비나·강의 등록 유도를 위한 무료 자료 (PDF 형태)
- 독자가 "이거 받으러 등록하고 싶다"는 느낌을 주는 것이 목적
- mystyle 연동 불필요 (정보 전달 위주, 문체보다 구조가 중요)

## Gamma 설정

| 항목 | 값 |
|---|---|
| 제작 방식 | `generate` |
| design_type | `document` |
| themeId | (없음 — Gamma 기본 테마 적용. 본인 테마 ID 있으면 여기에 추가) |
| templateId | (없음 — 전용 템플릿 있으면 여기에 추가하고 제작 방식을 `generate_from_template`으로 변경) |
| 언어 | `ko` |
| dimensions | `a4` |

## 리드마그넷 구성 원칙

- 분량: 5~10페이지 (너무 길면 안 읽힘)
- 핵심 가치를 빠르게 전달하고, 더 알고 싶으면 다음 단계로 연결
- 제목은 구체적인 결과물을 약속하는 형태
- 워크시트·체크리스트·템플릿 형식이 가장 효과적

## 유형별 예시

### 워크시트형
- 독자가 직접 채워 넣을 수 있는 빈칸 포함

### 체크리스트형
- Yes/No or 체크박스 형태

### 템플릿형
- 바로 복사해서 쓸 수 있는 형태

## 리드마그넷 구조

```
표지 — 제목 + 부제 + 저자·브랜드명
→ 이 자료를 활용하는 방법 (1페이지, 간략히)
→ 핵심 내용 (워크시트 / 체크리스트 / 템플릿)
→ 마지막 페이지 — 다음 단계 CTA
```

## CTA 작성 원칙

- "더 알고 싶으시면" 식의 약한 CTA보다 구체적 행동 유도
- 예: "이 내용을 직접 실습하고 싶으시다면 → [무료 웨비나 등록]"

## Gamma 생성 시 inputText 작성 요령

- 페이지별 구성 요소를 명확히 지정해서 전달
- textMode: `preserve` (구조 그대로 유지)
- cardSplit: `inputTextBreaks`
- 이미지 소스: `placeholder` (워크시트·체크리스트는 이미지보다 레이아웃이 중요)
- textOptions.amount: `brief` (핵심만)

## 주의사항

- 리드마그넷은 "미끼"가 아니라 "가치 선제공"이 원칙
- 내용이 너무 얕으면 신뢰도 하락 → 실제로 쓸 수 있는 수준의 자료여야 함
- 표지에 저자·브랜드 명시
