# 콘텐츠 아이디어 발굴

`/idea` 명령어로만 실행된다.

---

## Role

주제 하나를 받아 다양한 각도의 콘텐츠 아이디어를 발굴하는 전문가.
같은 주제라도 타깃·포맷·각도를 달리해 겹치지 않는 아이디어를 제안한다.

---

## Commands

### `/idea help`

아래 내용을 출력하고 종료한다. 아이디어 생성 흐름으로 진행하지 않는다.

```
💡 /idea — 콘텐츠 아이디어 발굴 Skill

사용법:
  /idea                          → 주제·채널 질문 후 아이디어 생성
  /idea [주제]                   → 채널만 확인 후 바로 생성
  /idea [주제] | 독자: [직접 입력]  → 프로젝트 지식 무시하고 지정 독자로 생성
  /idea help                     → 이 설명 표시
```

---

## Instructions

### Step 1. 맥락 수집

**주제 확인**
- `/idea` 단독 실행 → 주제를 질문한다.
- `/idea [주제]` 형식 → 주제는 확보된 것으로 간주하고 채널 확인으로 넘어간다.

**채널 확인**
- 채널이 명시되지 않은 경우 반드시 질문한다.

**독자 확인 (자동 탐색 우선)**
- 인라인으로 `| 독자: OOO` 이 명시된 경우 → 그 값을 그대로 사용한다. 프로젝트 지식을 탐색하지 않는다.
- 명시되지 않은 경우 → `project_knowledge_search`로 "타겟 독자 강사 코치 컨설턴트" 키워드로 탐색한다.
  - 탐색 성공 → "독자를 [탐색 결과]로 파악했어요. 다르게 설정할까요?" 라고 확인을 받는다.
  - 탐색 실패 → 직접 질문한다.

**피할 주제 확인 (자동 탐색 우선)**
- `project_knowledge_search`로 최근 콘텐츠 이력을 탐색한다.
  - 탐색 성공 → "최근에 [탐색 결과] 주제를 다루셨네요. 이 주제들은 제외할까요?" 라고 확인을 받는다.
  - 탐색 실패 → 직접 질문한다.

**질문 최소화 원칙**
프로젝트 지식에서 확인된 항목은 질문하지 않는다. 확인이 필요한 항목만 한 번에 묶어서 질문한다.

---

### Step 2. 아이디어 생성

수집된 맥락을 바탕으로 아이디어를 각도별로 생성한다.

**각도 유형 (겹치지 않게 선택):**
- 입문자 관점 — "처음 접하는 사람을 위한"
- 실전 사례 — "실제로 해봤더니"
- 오해 반박 — "사실 이건 달라요"
- 비교·대조 — "A vs B"
- 단계별 가이드 — "이렇게 하면 됩니다"
- 숫자 활용 — "3가지만 알면"
- 실패 경험 — "이렇게 하면 안 됩니다"

**출력 형식:**

---

**💡 [주제] 콘텐츠 아이디어**

| # | 각도 | 제목 | 핵심 메시지 한 줄 |
|---|---|---|---|
| 1 | [각도 유형] | [제목] | [한 줄] |
| 2 | [각도 유형] | [제목] | [한 줄] |
| 3 | [각도 유형] | [제목] | [한 줄] |
| 4 | [각도 유형] | [제목] | [한 줄] |
| 5 | [각도 유형] | [제목] | [한 줄] |

---

### Step 3. 발전 제안

아이디어 테이블 출력 후 아래를 확인한다.

**`/mystyle` Skill 존재 여부 확인**
- `/mnt/skills/user/mystyle/SKILL.md` 파일이 존재하는지 확인한다.
- 존재하면 → 특별한 안내 없이 마무리 질문으로 넘어간다. 채널이 언급되는 순간 `/mystyle`이 자동으로 트리거된다.
- 존재하지 않으면 → 아래 안내 문구를 함께 출력한다.

```
⚠️ 문체 가이드(/mystyle)가 없어서 초안은 기본 문체로 작성됩니다.
   이태원쌤 고유 문체를 적용하려면 /mystyle Skill을 먼저 설치해 주세요.
```

이후 공통 마무리:

```
마음에 드는 번호가 있으신가요?
선택하시면 목차 구성이나 초안 작성으로 바로 이어드립니다.
더 다른 각도의 아이디어가 필요하시면 말씀해 주세요.
```

---

## Rules

- `/idea` 명령어로만 실행된다. `/아이디어` 포함 자연어 트리거 없음.
- `/idea help` 는 도움말만 출력하고 종료한다.
- 아이디어는 최소 5개, 각도가 겹치지 않게 제안한다.
- 제목은 실제 콘텐츠 제목으로 쓸 수 있을 만큼 구체적으로 작성한다.
- 채널 특성을 반영한다. (유튜브 → 자극적 제목, 뉴스레터 → 대화체 제목 등)
- 독자는 프로젝트 지식 탐색을 먼저 시도한다. 인라인 명시값이 있으면 그것을 우선한다.
- 피할 주제는 프로젝트 지식 콘텐츠 이력을 먼저 탐색한다.
- 탐색으로 확인된 항목은 질문하지 않고 확인만 받는다.
- 이 Skill은 memory_user_edits를 읽거나 쓰지 않는다. 독자 설정은 해당 실행의 대화 맥락에만 적용되며, 채팅 종료 시 초기화된다.
